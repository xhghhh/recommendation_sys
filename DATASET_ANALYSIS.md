# TAAC2026 Demo Dataset 深度分析报告

## 1. 数据集概述

### 1.1 基本信息
| 属性 | 值 | 说明 |
|------|-----|------|
| **样本数** | 1,000 | 用户-物品交互记录 |
| **特征列** | 120 | 涵盖用户、物品、序列等多维特征 |
| **文件格式** | Parquet | 高效列式存储 |
| **文件大小** | ~39 MB | 包含丰富的序列特征 |

### 1.2 核心特点
- **冷启动场景**: 每个用户仅有1条记录
- **类别不平衡**: 正负样本比例约 7:1 (label_type 1:87.6%, 2:12.4%)
- **多模态特征**: 标量、数组、序列混合
- **跨域行为**: 4个Domain的行为序列

---

## 2. 特征结构深度解析

### 2.1 特征分类统计

```
┌─────────────────────────────────────────────────────────┐
│                    120 列特征分布                        │
├─────────────────────────────────────────────────────────┤
│  ID & Label          │  5 列 (4.2%)   │ 核心标识与标签  │
│  User Int Features   │ 46 列 (38.3%)  │ 用户整数特征    │
│  User Dense Features │ 10 列 (8.3%)   │ 用户浮点特征    │
│  Item Int Features   │ 14 列 (11.7%)  │ 物品整数特征    │
│  Domain Sequence     │ 45 列 (37.5%)  │ 跨域序列特征    │
└─────────────────────────────────────────────────────────┘
```

### 2.2 数据类型分布

| 数据类型 | 列数 | 占比 | 说明 |
|----------|------|------|------|
| **object** (list/array) | 67 | 55.8% | 序列和数组特征 |
| **float64** | 47 | 39.2% | 数值特征（含NaN） |
| **int64** | 5 | 4.2% | ID和时间戳 |
| **int32** | 1 | 0.8% | label_type |

**关键发现**: 超过一半的特征是数组/序列类型，需要特殊的预处理方法。

## 3. 各类特征详细分析

### 3.1 ID & Label 特征 (5列)

| 列名 | 类型 | 唯一值 | 说明 |
|------|------|--------|------|
| `user_id` | int64 | 1,000 | 用户唯一标识 |
| `item_id` | int64 | 837 | 物品唯一标识 |
| `label_type` | int32 | 2 | 1=点击(87.6%), 2=转化(12.4%) |
| `label_time` | int64 | 883 | 行为发生时间戳 |
| `timestamp` | int64 | 782 | 数据采集时间戳 |

**时间窗口**: 数据集中在约883秒内（约15分钟）

### 3.2 User Int Features (46列)

#### 标量型 (35列)
- 命名: `user_int_feats_{1,3,4,48-59,82,86,92-109}`
- 特点: 大部分为float64（存在缺失值）
- 示例值: 1753.0, 6.0, 42.0 等

#### 数组型 (11列)
- 命名: `user_int_feats_{15,60,62-66,80,89-91}`
- 特点: list<int64>，长度不一
- 示例: `[928, 556, 538, 739, 94]`

### 3.3 User Dense Features (10列)

- 命名: `user_dense_feats_{61-66,87,89-91}`
- 类型: list<float>
- 用途: 用户稠密向量表示
- 示例: `[0.1086, -0.0205, 0.0929, ...]`

### 3.4 Item Int Features (14列)

#### 标量型 (13列)
- 命名: `item_int_feats_{5-10,12-13,16,81,83-85}`
- 特点: 物品属性特征

#### 数组型 (1列)
- 命名: `item_int_feats_11`
- 示例: `[2980, 2770]`

### 3.5 Domain Sequence Features (45列) ⭐核心特征

这是本数据集最具价值的部分，包含4个行为域的序列：

```
Domain A (9列):  domain_a_seq_{38-46}
Domain B (14列): domain_b_seq_{67-79,88}
Domain C (12列): domain_c_seq_{27-37,47}
Domain D (10列): domain_d_seq_{17-26}
```

#### 序列特征内容分析

每个序列列包含多维度信息：

| 维度 | 示例值 | 说明 |
|------|--------|------|
| **时间戳** | `[1772695320, 1772626560, ...]` | 行为发生时间 |
| **行为类型** | `[2, 2, 7, 7, 2, ...]` | 不同行为编码 |
| **类别ID** | `[6, 6, 11, 11, 6, ...]` | 物品类别 |
| **数值统计** | `[840, 840, 0, 143, 215, ...]` | 累计数值 |
| **ID序列** | `[0, 0, 0, 764162, 0, ...]` | 相关ID |

#### 序列长度分布

根据分析，序列长度具有以下特点：
- **最短**: 0（无行为记录）
- **最长**: 16+（活跃用户的丰富历史）
- **平均**: 约5-8个行为

### 6.2 模型选择建议

| 模型 | 适用场景 | 预期AUC |
|------|----------|---------|
| **Logistic Regression** | 快速Baseline | 0.60-0.70 |

3. **特征交叉**
   - 用户-物品特征交叉
   - 跨Domain行为关联

## 9. Baseline 实验结果分析

我们在该数据集上进行了三个 Baseline 模型的实验：逻辑回归 (LR)、XGBoost 和 LightGBM。

### 9.1 实验设置

| 设置项 | 值 |
|--------|-----|
| 训练集 | 700条 (70%) |
| 验证集 | 100条 (10%) |
| 测试集 | 200条 (20%) |
| 特征维度 | 207维（经特征工程后） |
| 评估指标 | AUC, LogLoss, Accuracy |

### 9.2 各模型性能对比

| Logistic Regression | 0.4091 | 0.6964 | 0.600 | 🥉 基准 |


### 9.3 详细分析

#### 🥇 XGBoost (最佳性能)

**配置**:
- n_estimators=100, max_depth=6
- learning_rate=0.1
- subsample=0.8, colsample_bytree=0.8

**表现**:
- 训练集 AUC: 1.0000 (⚠️ 过拟合)
- 验证集 AUC: 0.5530
- **测试集 AUC: 0.7127** ✅

**分析**:
- 强大的非线性拟合能力
- 训练集完美拟合，但验证集下降明显 → 存在过拟合
- 测试集表现最好，泛化能力相对较强
- **建议**: 增加正则化，减小max_depth，或使用早停

#### 🥈 LightGBM (平衡选择)

**配置**:
- n_estimators=100, num_leaves=31
- learning_rate=0.1
- class_weight='balanced'

**表现**:
- 训练集 AUC: 0.9974
- 验证集 AUC: 0.4792
- **测试集 AUC: 0.6482**

**分析**:
- 训练速度快，内存占用低
- 同样存在过拟合问题
- 测试集表现中等，但Accuracy较高(0.805)
- **建议**: 调整num_leaves，增加正则化参数

#### 🥉 Logistic Regression (基准模型)

**配置**:
- C=1.0, penalty='l2'
- class_weight='balanced'
- 特征选择: Top 50

**表现**:
- 训练集 AUC: 0.6497
- 验证集 AUC: 0.4337
- **测试集 AUC: 0.4091** ❌

**分析**:
- 线性模型，无法捕捉复杂模式
- 特征选择后信息损失较大
- 存在欠拟合问题
- **建议**: 增加特征交叉，或使用多项式特征

### 9.4 关键发现

#### 1. 过拟合问题普遍存在

```
训练集 vs 测试集 AUC 差距:
XGBoost:  1.0000 → 0.7127  (差距 0.2873) ⚠️ 严重
LightGBM: 0.9974 → 0.6482  (差距 0.3492) ⚠️ 严重
LR:       0.6497 → 0.4091  (差距 0.2406) ⚠️ 中等
```

**原因分析**:
- 样本量小 (1000条)
- 特征维度高 (207维)
- 树模型容易过拟合

#### 2. 类别不平衡影响

- 正样本(转化): 12.4%
- 负样本(点击): 87.6%

**影响**:
- Accuracy 参考价值有限（预测多数类即可达到87.5%）
- **AUC 是更可靠的指标**

#### 3. 特征工程效果

- 原始特征: 120列
- 处理后特征: 207列
- 数组/序列特征池化后维度增加

### 9.5 改进建议

#### 短期优化

1. **缓解过拟合**:
   ```python
   # XGBoost
   xgb.XGBClassifier(
       max_depth=3,          # 减小深度
       min_child_weight=5,   # 增加最小叶子权重
       reg_alpha=0.1,        # L1正则
       reg_lambda=1.0,       # L2正则
   )
   ```

2. **交叉验证**:
   - 使用5折交叉验证替代单折划分
   - 更稳定的性能评估

3. **特征选择优化**:
   - 尝试不同的k值
   - 使用模型重要性筛选

#### 长期优化

1. **深度学习模型**:
   - DeepFM: 自动特征交叉
   - DIN/DIEN: 序列建模
   - 预期AUC提升: 0.05-0.15

2. **序列特征充分利用**:
   - 当前仅做简单池化
   - 使用Attention机制
   - 考虑时序信息

3. **集成学习**:
   - 多模型融合
   - Stacking策略
